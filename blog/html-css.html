The difference between CSS inline and inline-block

The difference between display inline and inline blocks: The display property controls how an element behaves in the page flow. Most elements are set to display block by default, which makes the elements stack on top of one another, even if there is space next to them. They have a width of 100% per default and will not flow horizontally even if the width is decreased.

Inline:

An element set to inline will only occupy the height and width of the element and the height and width properties will not have any effect. Contrary to the display block, inline does not force elements to a new line, but instead the element is positioned next to the existing content horizontally. Vertical padding and margins does not affect the inline elements layout. Some elements are set to inline by default such as anchors, bold text, italic and spans, this is to ensure that text elements are aligned to each other and do not have weird margins and heights affecting readability. It is good to note that an element with display inline cannot contain elements within it.

Inline-block:

An element set to inline-block will flow to the same line as other content, and will only occupy the height and width of the element (it does not default to 100%) similarly to inline. But these can have set width and height, meaning you can change their dimensions. They also support vertical and horizontal margins and padding. Some elements are set to inline block by default such as and . An element set to inline-block can contain elements within it contrary to inline.

In conclusion, both inline and inline-block allow content to flow next to each other and only take the space it occupies, but inline-block allows to set specific height, width, margins and padding to an element.
